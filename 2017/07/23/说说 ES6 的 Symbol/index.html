为什么会有 Symbol，它解决了什么问题？

每个 Symbol 都是独一无二的，可以用来避免冲突。这也是 Symbol 设计的初衷。考虑下面的情况，对应一个已经存在的对象（比如引用了一个第三方的库），如果想在其上边增加一个属性，可能带来下面几个问题：

  • 属性冲突
    新增的属性名已经存在，导致属性冲突
  • 可能影响用到属性枚举的代码
    如果库中有枚举属性的代码，比如 for-in 或 Object.keys()，新增加的属性也会被枚举

如果使用 Symbol 作为属性名，就可以避免冲突。

除此之外，Symbol 在一定程度上可以实现数据的私有化，因为常见的对象属性枚举方法，比如 for-in、Object.keys(obj) 和 Object.getOwnPropertyNames(obj) 都会过滤掉对象的 Symbol 属性。

但是，这也不是绝对的，可以通过 Object.getOwnPropertySymbols(obj) 和 Reflect.ownKeys(obj) 可以枚举对象中的 Symbol 属性。

关于对象属性的枚举，见另一篇文章。

注意事项

  • 不能使用 new 创建 Symbol

  • Symbol 不能被隐式转化为字符串

  • 可以显式的转换为字符串
