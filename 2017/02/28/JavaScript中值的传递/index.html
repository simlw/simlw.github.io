JavaScript中值的传递

Feb 28, 2017 | 阅读

JavaScript 中有两种类型的值，原始数据类型和引用数据类型。在《JavaScript高级程序设计》中提到，ECMAScript 中所有的函数参数都是按值传递的。为什么会这样？引用类型的值也是按值传递的？要理解这个问题，关键是需要理解 JavaScript 数据在内存中是如何存储的。

不同类型的值在内存中的存储

  • 原始数据类型
    原始数据类型的值，直接存储在栈内存中，解释器可以字节从内存（栈内存）中取到。如下图所示：

  • 引用类型的值
    引用类型的值，值由栈内存和堆内存中的数据组成，栈内存中存放的是值的引用（在堆内存中的地址），堆内存中存放着实际的值。如图：

变量的复制

变量的复制，实质上可以任务是变量多代表的值的复制。基本类型的值和引用类型的值，在复制的时候，有所区别。

  • 基本类型值的复制

    变量 | 值
    —|—
    ···|···
    a | 5

    变量 | 值
    —|—
    ···|···
    a | 5
    b | 5
    从上面可以看出，基本类型值的复制会在栈内存中分配一个新的内存区域，然后将复制之后新的变量和值分配到该内存上，之后a, b两个变量相互独立，操作任何一个都不会影响另一个。

  • 引用类型值的复制
    有了上面图中引用类型的内存示意图，下面看下，复制一个引用类型的对象，是怎么发生的。

    栈内存
    变量|值
    —|—
    ···|···
    a | 堆内存地址xxx

    栈内存
    变量|值
    —|—
    ···|···
    a | 堆内存地址xxx
    b | 堆内存地址xxx
    上面的例子说明，在引用类型的值复制的时候，实际是在栈内存中，将值的引用地址，按照原始类型值的方式复制了一份，复制之后的a, b两个变量，指向了堆内存中的同一个对象，这意味着如果修改 b 的属性，a 的值会受到影响。

按值传递

有了上面关于内存的知识，再来理解一下 函数参数都是按值传递的 这句话。
对于原始类型的值，相当于把实参复制给了形参，容易理解。
但是当参数是对象的时候，参数按照按值传递的方式，将栈内存中对象的引用复制给了形参，所以这个引用的传递实质是值的传递。
